# Grokへ渡す文脈

第二意見の質は、情報量より「何を判断してほしいか」と「どこを見れば確かめられるか」で決まる。

## 必ず入れるもの

1. **目的**: 最後に何を決めたいか。
2. **依頼**: レビュー、原因調査、設計比較など、Grokにしてほしいこと。
3. **対象**: リポジトリ、比較元、ブランチ、コミット、ファイル、行など。
4. **背景**: 判断に必要な利用場面や前提だけ。
5. **制約**: 変えてはいけないこと、言語、互換性、安全面、期限など。
6. **確認済み**: 実行したテスト、読んだ仕様、分かった事実。
7. **迷っている点**: まだ確かでないこと。暫定案があれば、反対意見も求める。
8. **出力形式**: 重要度、根拠の場所、理由、直し方など。

## リポジトリのレビューで伝えるもの

- 作業ディレクトリ
- 比較する差分。例: `origin/main...HEAD`。実行時は`--diff-base origin/main`で読み取り用コピーを作る
- 未コミット変更を含めるか。含める場合は`--include-uncommitted`を付ける
- 特に見てほしいファイルと観点
- すでに通った確認コマンド
- 「読み取り専用で、変更しない」という指示

差分を丸ごとプロンプトへ貼る必要はない。`run.sh --diff-base <比較元>`を使うと、ブランチ、状態、変更ファイル、差分を`/tmp`へコピーし、その場所をプロンプトへ追記する。未コミット変更も見る場合は`--include-uncommitted`を付ける。Grokにはshellを渡さないため、差分取得はラッパー側で行う。

## 渡さないもの

- 関係のない会話履歴やファイル
- 同じ説明の繰り返し
- 切り分け前の巨大なログ
- 秘密情報や対象外の個人情報
- 「この案が正しいはず」のような結論の押し付け

## テンプレート

```markdown
# 目的
最終的に決めたいこと。

# 依頼
Grokに確認してほしいこと。反例や見落としも求める。

# 対象
- cwd: /path/to/repository
- 差分: origin/main...HEAD（実行時に`--diff-base origin/main`を指定）
- 特に見るファイル: path/to/file
- 未コミット変更: 含めない

# 背景
判断に必要な利用場面と前提。

# 制約
- 読み取り専用。ファイルを変更しない
- 対象外の変更は提案しない
- 根拠のない推測は事実と分ける

# 確認済み
- 実行したコマンドと結果
- 参照した仕様

# 迷っている点
まだ確かでない点。暫定案は暫定だと明記する。

# 出力
重要な指摘から順に、重要度、ファイルと行、理由、直し方を示す。
問題がなければ、問題なしと残る不確かさを分けて書く。
```

## 用途ごとの追加情報

### 設計相談

- 選択肢と、それぞれで守りたいこと
- 現在の暫定案と、その理由
- 運用や将来の変更で困りそうな点

### 不具合調査

- 期待した動きと実際の動き
- 再現手順と毎回起きるか
- 試したことと、その結果
- 原因候補を事実と推測に分ける

### 内容レビュー

- 読者と、その時点で知っている前提
- その内容で身につけてほしい判断
- 正確さ、分かりやすさ、問いと答えの対応のどれを重く見るか
