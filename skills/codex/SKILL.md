# Codex

`codex` CLI を**セカンドオピニオンの相棒**として使う。

## 実行環境による使い分け

- 現在の実行主体が Codex の場合は、ユーザーが `codex exec` を明示的に指定していない限り、外部プロセスを起動せず、組み込みの `agent` / subagent を使う。
- Claude など Codex 以外のエージェントでは、以下の `codex exec` の自律起動条件を適用する。

## codex exec（外部エージェント・明示依頼時）

### いつ使うか

**原則: 一度推論・実装・デバッグを試みて解決できなかった場合は、積極的にセカンドオピニオンを得る**

**自律起動する条件**:
- 同じ問題を複数のアプローチで試したが解決できなかった
- デバッグしたが原因を特定できない
- 設計判断で迷っており、複数の視点から検討したい
- 長考が必要だと判断した場合

**ユーザーからの明示的依頼**:
- 「codexに相談して」「codexに聞いて」「codexならどう思う？」など、相談したい旨の表明

**使わない条件**:
- 初見で一度も試していない問題（まず自力で試す）
- 単純・定型的な作業（置換、フォーマット等）
- ワンステップで解決できると見込める場合

### 基本構文

```bash
codex exec "プロンプト"
```

### 相談の仕方

execに投げる時は、Claudeが試みた内容と詰まった点を添えると精度が上がる。

```bash
# デバッグ相談
codex exec "〇〇の原因を調べてほしい。△△を試したが解決しなかった。"

# 設計相談
codex exec "〇〇の設計についてどう思うか？自分は△△を考えているが迷っている。"

# 調査依頼
codex exec "〇〇の実装を調査して、〜の観点でまとめてほしい。"
```

### 結果の扱い

- execはファイル変更を行う可能性がある。**変更内容は必ず確認してから採用する**
- セカンドオピニオンとして参考にする。そのまま鵜呑みにしない
- バックグラウンド実行（`run_in_background: true`）を推奨

## codex review（明示的指示のみ）

ユーザーが「レビューして」「セルフレビュー」と明示的に言った時のみ実行する。自律的には起動しない。

### モード (排他: 1つだけ選択)

| モード | コマンド | ユースケース |
|--------|---------|-------------|
| ブランチdiff | `codex review --base <branch>` | PR前のセルフレビュー |
| 未コミット変更 | `codex review --uncommitted` | 実装途中のセルフチェック |
| 特定コミット | `codex review --commit <SHA>` | 特定コミットのレビュー |
| カスタムプロンプト | `codex review "レビュー指示"` | 自由な観点でレビュー |

指定がなければ `--base develop` を使用する。

### 結果の扱い

1. **指摘内容を要約**してユーザーに報告
2. **対応要否を判断**: 誤検知や軽微な指摘はスキップ
3. **対応する場合**: 修正を実施し、再度品質チェックを実行

## 共通オプション

| オプション | 説明 |
|-----------|------|
| `--title <TITLE>` | サマリーのタイトル（review のみ） |

# モデル一覧確認
jq -r '.models[] | "\(.slug)\t\(.description)"' ~/.codex/models_cache.json

## 実行時の注意

認証エラー（`token_expired` / `invalid_grant` 等）が出た場合のみ `! codex login` でユーザーに再ログインを案内する。

### その他の注意

- **cwdはプロジェクトルート（gitリポジトリ）で実行する**: codexはtrusted directoryチェックがあり、gitリポジトリ外から実行すると `Not inside a trusted directory` で失敗する。`~/.claude/skills/` 等のサブディレクトリに `cd` してから実行しないこと

## 禁止事項

- **モデルを指定しない**: codex のデフォルトモデルは常に最新モデルになるため、明示的にユーザーから指定された場合のみモデルを指定する。指定する場合のオプションは `-c model="<MODEL>"` （`-m` ではなく）。
