# 主張台帳テンプレート

根拠のない主張が文章に漏れ込むのを防ぐために使う。

```text
| ID | 主張 | 種類 | 根拠 | 読み手に必要か | リスク | 表現ルール |
|---|---|---|---|---|---|---|
| C1 |  | 事実 / 推論 / 意見 / 提案 / 不明 |  | yes/no |  |  |
```

## 表現ルール

### 事実

直接根拠がある場合だけ使える。

```text
APIキーがない場合、このエンドポイントは 401 を返す。
```

### 推論

根拠と、やわらかい表現が必要。

```text
認証処理は `AuthProvider` に分離されているため、APIキー形式の変更は主にこのモジュールへ影響すると考えられる。ただし、すべての呼び出し経路ではまだ検証していない。
```

### 意見

評価であることが見える必要がある。

```text
このプロジェクトでは、ファイル数を最小化することよりレビューしやすさが重要なため、このトレードオフを優先する。
```

### 提案

意図する行動と理由を含める。

```text
初回連携をシンプルかつ監査しやすく保つため、パートナー向けリクエストには API キー認証を使う。
```

### 不明

事実として本文に入れない。

```text
不明: パートナーAPIのレート制限は未確認。
```
