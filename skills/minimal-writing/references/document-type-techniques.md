# 媒体別プロファイル

普遍ルールは `SKILL.md` に置き、ここでは媒体ごとの差分だけを扱う。

どの媒体でも共通して、次を先に確認する。

- 読み手は誰か。
- 読後に何を理解・判断・実行してほしいか。
- 本文に必要な事実は何か。
- 推論してよい範囲はどこまでか。
- 不明点として残すべきものは何か。
- 削っても読み手の理解・判断・行動が変わらない文はどれか。

## 仕様書

主目的: 曖昧さを最小化し、実装とレビューを可能にする。

重視するもの:

- 対象範囲と非対象範囲
- 要件と制約の分離
- エラーケースとエッジケース
- データモデルまたはインターフェース変更
- 未決事項
- 判断理由

避けるもの:

- 美文調の表現
- 汎用的な重要性の説明
- 根拠のない保守性・拡張性アピール
- 未決定の挙動を確定事項のように書くこと

## ADR

主目的: なぜその判断をしたかを将来の読み手が追えるようにする。

重視するもの:

- 決定を1文で言えること
- 判断に必要な背景
- 検討した代替案
- 採用しなかった選択肢と理由
- 良い結果と副作用
- 見直し条件

避けるもの:

- 決定ではなく調査メモになっている状態
- 反対案を弱く書きすぎること
- 結果の悪い面を消すこと

## 技術ブログ

主目的: 経験から再利用可能な学びを渡す。

重視するもの:

- 問題設定
- 環境や前提
- 試したこと
- 結果
- なぜうまくいったか、またはうまくいかなかったか
- 制約
- 読み手が再利用できる知見

避けるもの:

- 1つの体験談を普遍的なベストプラクティスにすること
- 未検証の性能・品質主張
- 「重要です」の連打
- すごそうだが再利用できない抽象論

## 社内提案文

主目的: 読み手が採否、優先順位、次の行動を判断できるようにする。

重視するもの:

- 結論と提案内容
- 背景にある問題
- 放置した場合の影響
- 提案の効果範囲
- コスト、リスク、代替案
- 決めてほしいこと

避けるもの:

- 危機感だけで根拠がない主張
- すべての問題を一度に解く提案
- 誰が何を判断すべきか分からない文章

## 職務経歴書

主目的: 読み手が、役割、判断、行動、結果、再現性を評価できるようにする。

重視するもの:

- 役割と責任範囲
- 状況、判断と行動、結果
- 本人が実際に担った範囲
- 数値や成果の定義
- チーム成果と本人貢献の切り分け
- 面接で説明できる根拠

避けるもの:

- `主導`、`責任者`、`顧客折衝`、`要件定義全体` などが実態より強く読まれること
- チーム成果を本人単独の成果に見せること
- 覚えていない実績や説明できない因果を載せること
- 会社・案件の公開範囲を超える情報を出すこと

エピソード設計が主目的なら `career-accountability-review` を優先する。この skill は、主張の根拠、過大表現、冗長さ、公開範囲の検品で併用する。

## 公開プロフィール

主目的: 不特定の読み手に、何を担える人かを安全な範囲で伝える。

重視するもの:

- ドメインや役割の抽象化
- 公開してよい粒度
- 案件名・会社名・内部指標の扱い
- 数字を出す必要性
- 読み手が次に見たい情報

避けるもの:

- 内部プロジェクト名や略称が目立ちすぎること
- 内部インシデント、セキュリティ詳細、正確すぎる指標の露出
- 説明コストの高い社内用語
- 自己評価だけで具体行動がない強み表現

## X/Twitter 投稿

主目的: 1つの主張、1つの緊張、1つの具体的な引っかかりを伝える。

重視するもの:

- 1投稿1主張
- 具体例または対比
- 読後に残る問いや余韻
- 必要な不確実性ラベル

避けるもの:

- 1投稿にブログ記事を詰め込むこと
- 複数テーマを混ぜること
- 正論だが引っかかりがない文章
- ユーザーの声を消しすぎること

## 論文・研究風文書

主目的: 範囲を限定した、再現可能な主張にする。

重視するもの:

- 問題定義
- 先行研究またはベースラインとの差分
- 方法
- 結果
- 解釈
- 制約
- 再現性に必要な情報

避けるもの:

- 結果が示す以上の主張
- 弱い根拠から因果を含意すること
- 否定的または不確かな結果を隠すこと
- 再現不能な説明

## README / 利用手順

主目的: 読み手が迷わず使い始め、必要な判断をできるようにする。

重視するもの:

- 何のためのものか
- 最短の利用開始手順
- 前提条件
- 代表的な使い方
- よくある失敗と対処
- 詳細情報への導線

避けるもの:

- 背景説明が長すぎて手順に到達しないこと
- すべての機能を先に説明すること
- 実行できない例

## メール / 連絡文

主目的: 読み手が状況、依頼、期限、必要な行動をすぐ判断できるようにする。

重視するもの:

- 結論
- 相手にしてほしいこと
- 期限
- 背景は必要最小限
- 添付・参照先
- 返信が必要かどうか

避けるもの:

- 礼儀のためだけの長い前置き
- 依頼内容が文末まで出てこないこと
- 相手の判断に不要な経緯
