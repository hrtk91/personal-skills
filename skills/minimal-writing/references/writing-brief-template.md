# 執筆ブリーフテンプレート

下書き前に使う。

```text
文書種別:
読み手:
読み手の目的:
書き手の目的:
読後に読み手が到達すべき状態:
使用してよい事実:
使用してよい推論:
禁止する主張・トーン:
必要なセクション:
長さの制約:
トーン:
不明点:
```

## 記入例: 仕様書

```text
文書種別: 仕様書
読み手: 実装者、レビュー担当者、将来の保守者
読み手の目的: 何を作るか、何を作らないか、どこに未決事項があるかを理解する
書き手の目的: 実装とレビューの手戻りを減らす
読後に読み手が到達すべき状態: 変更を実装できる、または具体的なブロッカーを挙げられる
使用してよい事実: チケット、リポジトリのコード、ユーザーが提供した会議メモ
使用してよい推論: 現在のモジュール構造から直接示される影響範囲
禁止する主張・トーン: 根拠のない将来の保守性主張、曖昧な「セキュリティ改善」主張
必要なセクション: 背景、目的、非目的、要件、制約、不明点、ADRリンク
長さの制約: エッジケースを保てる範囲でできるだけ短く
トーン: 正確、中立、実装寄り
不明点: 外部APIのレート制限、リトライ回数、移行日
```
