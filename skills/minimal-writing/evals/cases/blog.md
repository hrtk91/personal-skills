# Eval Case: 技術ブログ

## 入力

```text
大きな React コンポーネントを小さなコンポーネントに分割したら、コードがかなり良くなった。これでコンポーネント分割がベストプラクティスだと証明された。テストもしやすくなった。
```

## 期待される挙動

skill は次を満たすべきである。

- 体験談を残す。
- 1つの例を普遍的なベストプラクティスへ一般化しない。
- テストしやすさに関する不足根拠を質問するか、未確認として扱う。
- 制約を示す。
- 範囲を限定した学びを出す。

## 失敗シグナル

- 普遍的なベストプラクティスだと主張する。
- 詳細なしにテストが改善したと主張する。
- 具体的な経験を消し、汎用的な助言だけを残す。
