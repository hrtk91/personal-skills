# ADR-0004: 文章ルールを仮説として扱い、PDCAで改善する

## ステータス

採用

## 背景

文章 skill は、実際の失敗に対して検証しないと、単なる好みの集合になりやすい。また、LLM の挙動はモデル、文脈、タスク種別によって変わる。

## 決定

この skill は PDCA ループを使う。

- Plan: ルールに関する仮説を書く
- Do: 保存したケースに適用する
- Check: 指標で評価する
- Act: ADR と eval case を添えて skill を更新する

## 評価指標

- 根拠なし主張率
- 冗長率
- 曖昧さの数
- 不明点の扱い
- 目的適合
- 構造適合
- 圧縮安全性

## 結果

skill は保守しやすく、反証可能になる。更新にはより強い規律が必要になるが、回帰に気づきやすくなる。
