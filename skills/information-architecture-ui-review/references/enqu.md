# EnQu向け情報設計リファレンス

## 商品導線

現在の商品方針は、EnQu全体を売るのではなく、テキストで設計説明を練習する「設計説明トレーニング β」を入口にすること。

```text
LP
  → 無料3問
  → 商品内容・価格の理解
  → 20問の固定デッキ
  → 回答・LLM採点
  → 不足点の理解
  → 復習・再回答
```

βの初期デッキは固定20問と進捗を中心にする。手動の「復習に追加」「今は外す」「もう一度解く」は次の段階で追加し、学習履歴から復習候補を自動編成する機能は利用データが溜まってから検討する。

## 画面ごとの主目的

| 画面 | 主目的 | 主操作 |
| --- | --- | --- |
| LP | 商品の価値と対象者を理解する | 無料問題を始める |
| 無料問題 | 説明練習と採点体験を得る | 回答を送信する |
| 商品案内 | 何が何問あり、何が得られるか理解する | 購入へ進む |
| デッキ | 残り問題と次の一問を把握する | 問題を始める |
| 問題 | 問いと回答条件を理解する | テキストで回答する |
| 採点結果 | 不足していた説明を理解する | 復習・次の問題へ進む |

## コンテンツの役割分担

- `learn` の紙芝居・解説: 概念を理解するための予習・復習。
- 設問・採点基準: 自分の言葉で説明できるか確認するための実践。
- βデッキ: 問題を選ぶ手間をなくし、説明練習を順序立てて完了するための体験。

同じ知識を使う場合でも、教材の説明をそのまま問題にしない。教材は理解を助け、問題は想起と説明を要求する。採点後に関連教材へ戻れる導線を設ける。

## EnQuのレビュー観点

- LPの主CTAが「無料3問を試す」に集中しているか。
- 問題画面で、問題文・回答欄・送信操作の関係が一目で分かるか。
- 採点結果で、スコアより先に「何が不足していたか」と「次に何をするか」が理解できるか。
- デッキ画面で、進捗・未回答・復習候補・次の一問が混ざっていないか。
- 購入前に、問題数、回答方法、採点内容、利用期限、再回答の扱いが分かるか。
- ダークなターミナル風表現を使う場合も、濃淡・余白・文字階層による可読性を優先する。
